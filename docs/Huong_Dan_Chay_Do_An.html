III. CÀI ĐẶT CƠ SỞ DỮ LIỆU

Bước 1 — Khởi động MySQL

Đảm bảo MySQL Server đang chạy. Với XAMPP, bật XAMPP Control Panel → Start MySQL.

Bước 2 — Tạo database và import schema

Mở MySQL Workbench hoặc terminal, chạy lần lượt:

# Đăng nhập MySQL mysql -u root -p # Tạo database (nếu chưa có) CREATE DATABASE housekeeping_service CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci; EXIT;
# Import schema (tạo 21 bảng, views, stored procedures) mysql -u root -p housekeeping_service < database/schema.sql # Import dữ liệu mẫu (25 users, 20 helpers, bookings, reviews...) mysql -u root -p housekeeping_service < database/sample_data.sql

Bước 3 — Kiểm tra dữ liệu

mysql -u root -p housekeeping_service SELECT COUNT(*) AS tong_helpers FROM helpers; -- Kết quả: 20 SELECT COUNT(*) AS tong_users FROM users; -- Kết quả: 25 SELECT COUNT(*) AS tong_services FROM services; -- Kết quả: 6
Thành công: Nếu kết quả trả về helpers = 20, users = 25, services = 6 thì cơ sở dữ liệu đã sẵn sàng.

IV. CÀI ĐẶT VÀ CHẠY BACKEND

Bước 1 — Cài đặt dependencies

cd backend npm install

Bước 2 — Tạo file .env

Sao chép file .env.example thành .env và điền thông tin thực tế:

cp .env.example .env

Mở backend/.env và chỉnh sửa các giá trị:

PORT=5000 NODE_ENV=development # Thông tin kết nối MySQL DB_HOST=localhost DB_PORT=3306 DB_USER=root DB_PASSWORD=your_mysql_password # ← Điền mật khẩu MySQL của bạn DB_NAME=housekeeping_service # Khóa bí mật JWT (đặt chuỗi ngẫu nhiên dài ≥32 ký tự) JWT_SECRET=your_jwt_secret_key_here JWT_EXPIRES_IN=15m JWT_REFRESH_SECRET=your_refresh_secret_key_here JWT_REFRESH_EXPIRES_IN=7d # URL frontend (dùng cho CORS) CLIENT_URL=http://localhost:5173

Bước 3 — Khởi động server

# Chế độ development (tự động reload khi thay đổi code) npm run dev # Hoặc chế độ production npm start
Thành công: Terminal hiển thị:
✅ Server đang chạy tại http://localhost:5000
✅ Kết nối MySQL thành công

IX. HƯỚNG DẪN SỬ DỤNG CÁC TÍNH NĂNG CHÍNH

1. Luồng đặt dịch vụ (Customer)

  1. Truy cập http://localhost:5173 → Trang chủ hiển thị 6 dịch vụ
  2. Chọn dịch vụ → Xem danh sách helper tại Hà Nội với rating và giá
  3. Nhấn "Đặt lịch" trên profile helper → Điền ngày giờ, địa chỉ
  4. Xác nhận booking → Hệ thống tính giá tự động tại backend
  5. Vào Lịch đặt để theo dõi trạng thái

3. Thanh toán & Đánh giá

  • Sau khi booking chuyển sang completed, customer vào chi tiết booking → nhấn "Xác nhận thanh toán"
  • Sau thanh toán, customer có thể đánh giá 1–5 sao kèm nhận xét
  • Helper nhận thông báo về thanh toán và đánh giá mới

4. Tính năng thông báo

  • Icon chuông 🔔 trên Navbar hiển thị số thông báo chưa đọc (badge đỏ)
  • Nhấn vào → Trang /notifications liệt kê tất cả thông báo
  • Nhấn vào từng thông báo để đánh dấu đã đọc
  • Nút "Đánh dấu tất cả đã đọc" ở góc phải

5. Quản trị Admin

  • Dashboard: thống kê tổng booking, doanh thu, users, helpers đang hoạt động
  • Quản lý Users: xem danh sách, kích hoạt/vô hiệu hóa tài khoản
  • Quản lý Booking: xem tất cả đơn hàng, lọc theo trạng thái
  • Duyệt Helper: xác minh tài khoản helper mới đăng ký

X. XỬ LÝ SỰ CỐ THƯỜNG GẶP

Lỗi kết nối MySQL

Error: connect ECONNREFUSED 127.0.0.1:3306

Giải pháp:

  • Kiểm tra MySQL đang chạy: sudo service mysql status
  • Kiểm tra thông tin trong backend/.env (DB_HOST, DB_USER, DB_PASSWORD)
  • Thử kết nối thủ công: mysql -u root -p

Lỗi CORS khi frontend gọi API

Access to XMLHttpRequest blocked by CORS policy

Giải pháp: Kiểm tra CLIENT_URL trong backend/.env khớp với URL frontend (http://localhost:5173).

Frontend không kết nối được backend

Network Error / ERR_CONNECTION_REFUSED

Giải pháp:

  • Đảm bảo backend đang chạy tại cổng 5000
  • Kiểm tra frontend/.env: VITE_API_URL=http://localhost:5000
  • Sau khi sửa .env, restart frontend: npm run dev

Token hết hạn — 401 Unauthorized

Giải pháp: Frontend tự động làm mới token qua /api/auth/refresh. Nếu vẫn lỗi, đăng nhập lại.

Lệnh nhanh khởi động lại toàn bộ dự án
